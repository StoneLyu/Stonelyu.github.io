Motivation for MapReduce

  • process lots of data(>1TB)
  • parallelize across
  • make this easy
    • automatic parallelization and distribution
    • fault-tolerance 容错 -> 给的数据集里部分数据有问题
    • I/O scheduling
    • status and monitoring

Embarrassingly parallel worklod

  • a general model:

  • definition of Map, Reduce and Job:

  • 在Hadoop中，每个MapReduce任务都被初始化为一个job,每个job又可分为两个阶段: map阶段和reduce阶段。这两个阶段分别用两个函数来表示。

    • map函数接收一个<key,value>形式的输入，然后同样产生一个<key,value>形式的中间输出，Hadoop会负责将所有具有相同中间key值的value集合在一起传递给reduce函数，
    • reduce函数接收一个如<key,(list of values)>形式的输入，然后对这个value集合进行处理，每个reduce产生0或1个输出，reduce的输出也是<key,value>形式。
  • 基本概念
    • Map: 映射，接受一个键值对(key-value pair)，产生一组中间键值对。MapReduce框架会将map函数产生的中间键值对里键相同的值传递给一个reduce函数. (map的过程中会自动剔除损坏的数据)
    • reduce: 归约，接受一个键，以及相关的一组值，将这组值进行合并产生一组规模更小的值（通常只有一个或零个值）。
    • shuffle: 一个map产生的数据，结果通过hash过程分区却分配给了不同的reduce任务.
    • job: 任务，分map端和reduce端

MapReduce Programming Model

Functional Programming Basics 函数式编程，这里主要是lambda函数

Functional Programming

  • definition: 它将电脑运算视为函数的计算。函数编程语言最重要的基础是λ演算（lambda calculus），而且λ演算的函数可以接受函数当作输入（参数）和输出（返回值）。

Big Data and Functional Programming

  • 函数式编程和并行编程非常适配

feature of FP 函数式编程的特性/功能

  • 不修改原有数据结构，直接创建新的数据对象，并一一映射过去
  • 高阶函数，把别的函数当成参数，并提供一个更简便的处理收集过程的方式
  • 在MapReduce中有两个非常有用的高阶函数：map and fold, or reduce
higher-order function: map

map函数把一个给定函数应用于列表中所有的元素，并以新列表的形式返回结果,利用map函数可以很轻松地执行并发函数。

higher-order function: ford/reduce

fold函数可以从一个初始值开始，将一个给定函数迭代地作用于列表中的所有元素。返回的结果是从初始值迭代应用给定函数到最后一个元素的值。

Reduce Function Parallel Excution
  • 一般来讲，reduce函数是不可以并行化的，因为列表中的所有元素在产出最后结果的过程中都需要被访问到。
  • 但在很多情况下，可以通过从子集中聚合(aggregate)局部结果来产出最终结果。
  • 问题是：我们要如何去把reduce函数的输入分割成子集，尤其是当这些输入是前序map函数的输出的情况下，如何实现？
programming model
  • input & output 是一系列的key-value 键值对

  • 程序员需要编写两个函数：

    • map
      • map(in_key, in_value) -> list(out_key, intermediate_value)
      • 处理输入的键值对，输出一个列表的介质对(out_key, intermediate_value)
    • reduce
      • reduce(out_key, list(intermediate_value)) -> list(out_key, out_value)
      • 把某个确定的key的所有介质值全部组合起来
    • 产出某一给定key 的一系列聚合输出值（通常只有一个）

  • 这里的key是用来分割和分组reduce函数的输入的，使之更有效率

the combiner function 组合器函数
  • 组合器是一个在map和reduce之间最小化数据传输的优化机制
  • 组合器函数在map端运行，合并部分map的媒介结果

Hadoop MapReduce Framework

hadoop basics

  • hadoop is a open source implementation of Google’s MapReduce Framework
  • hadoop is writtten in java
  • hadoop can run on a single machine
    • standalone model
    • pseudo-distribute mode

HDFS

  • HDFS 是一个分布式文件系统，
  • 由一个节点的集群组成
    • 其中一个节点由特殊的角色，称为name node
    • 其他所有的节点称data nodes，用于存储文件
    • HDFS在主文件系统的特定位置存放文件
  • HDFS被设计来存大尺寸文件
    • 大文件会被分成固定大小的小块，如128M
    • 小块被存在不同的数据节点下，这些小块可以重复
    • name node存放元数据，元素据会记录文件是由哪些块组成

YARN Framework -> 一个通用的资源管理框架 -> 另一种资源导航器

YARN concept

  • YARN是应用请求资源
  • resource manager 在集群的一个特殊节点上运行
  • 每一个节点都有node manager运行
  • 每一个application 都有一个 ApplicationMaster
    • 代表应用向集群请求资源
    • 监控应用的执行情况
    • 框架细节
  • 资源请求 resource request
    • 主要是内存和CPU；主要依靠配置好的调度器
  • 容器 container
    • 资源以容器的形式定位
    • 如：Java Process
  • 默认配置
    • 一个node manager有四个核心，每一个运行两个容器
    • node manager为YARN提供8G内存，每一个容器有2G
    • 理论上一个节点可以运行8个容器
