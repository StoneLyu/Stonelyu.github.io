MongoDB 进阶操作，包括索引、底层细节、mongoose库的使用等

mongoDB indexing

indexing 索引

  • 在某个collection中的某个字段A的索引是一个可以在A中更有效率地找到需要的值的数据结构。

  • 一个索引包括：

    1. additional properties can be specified for an index. 创建索引的时候可以加入其他的限制性属性
      • sparseness, uniqueness, background
  • most MongoDB indexes are organized as B-Tree structure

simgle field index

  • db.users.find({score: {"$lt": 30}}).sort({score: -1});
    1. users: collection
    2. {score: {"$lt": 30}}: query criteria
    3. {score: -1}: sort order

compound index (复合索引)

  • 复合索引是简单索引结构存放指向多个字段的参考的collection
  • 复合索引的字段的顺序非常重要：
    1. 索引是按第一个字段，第二个字段。。。以此类推的顺序进行排序的
    2. 支持：
      • db.users.find({userid: "ca2", score: {$gt: 30}});
      • db.users.find({userid: "ca2"});
    3. 不支持：
      • db.users.find({score: 75});

database – data layer

web applications – database

  • database tier in multi-tier(N-tier) application architecture
    1. 为当前应用存储数据
    2. CRUD operation 增删改查
  • database server/ DBMS
    1. RDBMS (MySQL)
    2. NoSQL DBMS (MongoDB)
    3. choice of DBMS is crucial(关键)
  • express integrates with many DBMS express整合了很多DBMS
    • MySQL, PostagreSQL, MongoDB, Redis and so on

database drivers

  • all database management systems works like a “server” application
    1. running on a host and waiting for connections from clients
      • simple command line and shell client
      • GUI shell client
      • program based client
    2. different protocals db server used to communicate with their clients
  • all database management systems provide language based drivers to allow developers to write client in varous language
    1. open/close connection
    2. translate between language specific construct(function, method) and database queries
    3. translate between language specific data types and database defined data types
  • MongoDB 提供很多原生驱动

higher level module/package

  • the native db drivers provide basic supports for client side programming
    • powerful, flexible, but usually not easy to use
  • higher level modules usually provide more convenient ways to communicate with db servers
    • mongoose

object data model/ object relational model

  • 与数据库的交互方式：
    1. 使用原生查询语句
    2. 使用object data model (ODM)/ object relational model (ORM)
  • 将网站应用数据表示成object，再映射到数据库里
    1. productivity 生产率：程序员更习惯于思考js对象而非数据库语法
    2. performance：mapping会比直接使用db semantics更慢
  • 节点支持多种ODM/ORM解决方案
    • Mongoose：异步环境

Mongoose basic concepts

  • schema：

一种以文件形式存储的数据库模型骨架，不具备数据库的操作能力。

schema模式: 一个定义collection中的documents的抽象数据结构

每一个name/value pair就是一个路径(path)

  • model:

由Schema发布生成的模型，具有抽象属性和行为的数据库操作对

model是编译过后的schema，model是collection绑定上schema

document: document是model的实例与collection中的真实的document相连

static methods

  • 如果常对某collection进行某操作，我们可以将该操作实现为static method 或者 instance method
  • static method 被定义在Model(collection)上，任何标准的query/aggregate 都可以被实现为static method
  • static method提高数据库相关代码的复用率和模块化程度

static method -example

instance method

  • instance method 定义在文件实例(document instance)上
  • 常被用于对给定的document创建queries

database connection

  • 开启和关闭数据库很花时间->尽量减少数据库的打开和关闭
  • 最好的解决办法是：所有的请求分享一个连接池，在应用关闭的时候关闭这个连接池
  • Mongoose会自动管理这个连接池

  • 不需要在app层面进行开闭操作

    • 连接数据库：
      mongoose.connect('mongodb://username:password@host:port/database?options...');
